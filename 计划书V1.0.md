## 1. 项目概述

智者是一个面向每日科技新闻收集、筛选、排序和摘要的智能体系统。它的核心目标不是简单堆砌新闻链接，而是帮助用户在短时间内理解当天科技领域真正重要的变化。

智者每天自动从多个科技信息源采集内容，包括科技媒体、RSS、Hacker News、GitHub、论文平台、公司博客、政策发布源等。系统会对内容进行清洗、去重、聚类、分类、摘要和重要性排序，最终生成一份结构清晰、重点明确、带有来源链接的每日科技简报。

## 2. 产品目标

1. 每天自动生成一份 5 分钟可读完的科技新闻简报。
2. 对新闻进行重点排序，让用户先看到最重要的内容。
3. 对每条重点新闻说明“发生了什么”“为什么重要”“可能影响谁”。
4. 保留来源链接，减少幻觉和误读。
5. 支持人工审核、置顶、隐藏、改写和权重调整。
6. 后续支持个性化关注领域，例如 AI、芯片、开发者工具、网络安全、开源项目、公司动态等。

## 3. 参考范式

智者可以参考目前常见智能体系统的组织方式：

1. 多智能体分工模式：参考 CrewAI 中 agents、tasks、crews 的设计，将采集、筛选、排序、摘要、校验、发布拆成不同角色。
2. 流程编排模式：参考 LangGraph 的有状态工作流，让每天的新闻处理流程可恢复、可追踪、可插入人工审核。
3. 工具调用模式：参考 OpenAI Agents SDK 的工具、护栏、handoff、tracing 思路，让模型负责判断和生成，让工具负责抓取、存储、发布。
4. 新闻摘要机器人模式：参考常见 AI newsletter、HN digest、RSS summarizer 的输出形态，但增加更强的优先级排序和影响分析。

## 4. 用户场景

### 4.1 每日晨报

用户每天早上收到一份科技简报，快速了解过去 24 小时内最重要的科技新闻。

### 4.2 重点追踪

用户关注 AI、芯片、开源工具、安全漏洞等主题，智者会自动提高相关内容权重。

### 4.3 决策辅助

当出现重大新闻，例如模型发布、法规变化、安全漏洞、公司战略调整时，智者不仅摘要内容，还解释潜在影响。

### 4.4 内容沉淀

每日新闻可以沉淀为 Markdown、Notion、GitHub Issue、网页或知识库，方便后续搜索和复盘。

## 6. 智能体角色设计

| 智能体角色 | 责任 | 输入 | 输出 |
|---|---|---|---|
| 采集者 Collector | 抓取多源新闻 | RSS、API、网页 | 原始候选新闻 |
| 清洗者 Cleaner | 清理无效内容 | 原始新闻 | 标准化新闻对象 |
| 聚类者 Clusterer | 去重和事件聚类 | 标准化新闻 | 新闻事件组 |
| 分类者 Classifier | 判断主题与实体 | 新闻事件组 | 分类和标签 |
| 排序者 Ranker | 计算新闻重要性 | 分类新闻 | 优先级列表 |
| 摘要者 Summarizer | 生成简明摘要 | 高优先级新闻 | 摘要和重点 |
| 校验者 Verifier | 检查来源和可信度 | 新闻、来源链接 | 校验结果 |
| 编辑者 Editor | 生成日报结构 | 排序新闻 | 日报草稿 |
| 发布者 Publisher | 分发内容 | 日报草稿 | 邮件/网页/消息 |
| 反馈分析者 Analyst | 学习人工调整 | 用户反馈 | 权重优化建议 |

## 9. 重点排序模型

新闻总分建议采用加权评分：

```text
总分 =
0.25 * 行业影响力 +
0.20 * 用户相关性 +
0.15 * 时效性 +
0.15 * 来源可信度 +
0.10 * 新颖性 +
0.10 * 可行动性 +
0.05 * 讨论热度
```

### 9.1 优先级定义

| 等级 | 含义 | 示例 |
|---|---|---|
| P0 | 今日必须知道 | 重大模型发布、严重安全漏洞、关键监管政策 |
| P1 | 高价值重点 | 大厂产品更新、重要开源项目、重要论文 |
| P2 | 可选阅读 | 普通融资、产品小更新、行业评论 |
| P3 | 观察归档 | 低可信、重复报道、营销稿 |

### 9.2 排序解释字段

每条重点新闻需要保留排序理由：

1. 为什么这条新闻重要。
2. 它影响哪些人或行业。
3. 它是否来自可靠来源。
4. 是否有多个来源交叉验证。
5. 是否值得用户采取行动。


## 16. Prompt 与模型策略

### 16.1 分类 Prompt

分类时要求模型输出结构化 JSON，包括主分类、标签、涉及公司、涉及技术、可信度提示。

### 16.2 排序 Prompt

排序时要求模型不要只看热度，还要解释影响力。输出必须包含：

1. 重要性分数。
2. 重要性理由。
3. 影响对象。
4. 是否需要进入今日最重要栏目。

### 16.3 摘要 Prompt

摘要时禁止泛泛而谈，要求：

1. 先说事实。
2. 再说影响。
3. 保留不确定性。
4. 不编造来源中没有的信息。

## 18. 风险与应对

| 风险 | 表现 | 应对 |
|---|---|---|
| 信息源噪声高 | 热点不等于重要 | 增加可信度和影响力权重 |
| 重复报道过多 | 日报内容冗余 | 做事件聚类和来源合并 |
| 摘要幻觉 | 模型编造不存在的信息 | 强制引用来源，保留原文链接 |
| 排序不符合用户偏好 | 重要新闻被漏掉 | 增加人工反馈和权重配置 |
| API 成本过高 | 每日调用过多 | 先规则过滤，再对高候选调用 LLM |
| 来源版权风险 | 复制大段正文 | 只保存摘要、链接和必要元数据 |
| 抓取失败 | 日报缺失 | 加重试、备用源、失败告警 |

## 19. MVP 验收标准

V1.0 的成功标准：

1. 每天稳定生成一份科技新闻日报。
2. 日报包含 10-20 条高质量新闻。
3. Top 5 新闻中至少 4 条被人工认为重要。
4. 每条 P0/P1 新闻都有来源链接。
5. 每条重点新闻都有“为什么重要”的解释。
6. 日报阅读时间控制在 5 分钟内。
7. 系统可以通过配置调整新闻源、栏目和排序权重。

## 20. 后续路线图

### V1.1 个性化

1. 根据用户阅读、收藏、隐藏行为调整权重。
2. 支持关注主题和屏蔽主题。
3. 支持不同用户收到不同日报。

### V1.2 深度分析

1. 对重大事件生成专题分析。
2. 自动追踪事件后续发展。
3. 形成周报和月报。

### V1.3 知识库

1. 将每日新闻沉淀为可搜索知识库。
2. 支持问答：某家公司最近发生了什么。
3. 支持趋势分析：某技术方向热度是否上升。

### V1.4 多渠道发布

1. 邮件 newsletter。
2. Notion 数据库。
3. 微信/飞书/Slack。
4. 静态网站或个人知识库。
